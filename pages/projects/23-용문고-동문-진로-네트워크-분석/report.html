23번 주제: 용문고 동문 진로 네트워크 분석

Designing an Alumni-Student Mentorship Matching System Using Network Analysis

※ 본 문서는 교육용 시범 보고서이며, 데이터는 개인정보를 포함하지 않는 시범용 가상 데이터입니다. 실제 운영 시에는 학생·동문 동의와 개인정보 보호 절차가 필요합니다.

1. 프로젝트 개요

이 프로젝트는 용문고 동문 네트워크가 후배들의 진로 탐색에 어떻게 기여할 수 있는지를 데이터로 분석하는 과제이다. 학생의 희망 전공, 관심 산업, 필요한 역량과 동문의 전공, 산업, 직무, 멘토링 가능 분야를 연결하여 동문-학생 진로 매칭 모델을 설계한다.

항목 내용
문제의식 동문 네트워크는 후배들의 진로 탐색에 어떻게 기여할 수 있는가
데이터 동문 직업군, 전공, 산업, 멘토링 가능 분야, 학생 관심 분야
분석 방법 네트워크 분석, 매칭 알고리즘, 군집분석, 상관분석
산출물 동문-학생 진로 매칭 모델, 산업별 멘토 수요-공급 격차표
연결 전공 경영학, 데이터사이언스, 네트워크 과학, HR

2. 문제 정의 카드

질문 답변
내가 본 문제는 무엇인가? 학생들은 진로 정보를 인터넷에서 많이 찾지만, 실제 직무 경험과 살아 있는 조언을 얻기 어렵다.
왜 이 문제가 중요한가? 진로 선택은 정보의 질에 따라 달라진다. 좋은 멘토를 만나는 학생과 그렇지 못한 학생 사이에 보이지 않는 격차가 생길 수 있다.
어떤 데이터를 사용할 것인가? 학생 희망 전공·직업 설문, 동문 전공·직업군·산업·멘토링 가능 분야, 커리어넷·워크넷 직업정보
어떤 분석을 할 것인가? 학생-동문 후보쌍을 만들고 산업·전공·직무·역량 적합도에 가중치를 주어 매칭점수를 산출한다.
누구에게 제안할 것인가? 학교, 동문회, 진로진학부, 학부모회, 용문포럼/용IT포럼
전공 관심과 어떻게 연결되는가? 데이터사이언스와 HR 매칭, 네트워크 과학을 활용해 사람 중심의 진로 지원 시스템을 설계한다.

3. 데이터 설계

변수명 설명 SPSS 측도 비고

alumni_industry 동문 종사 산업 명목 동문이 현재 일하는 산업 분야

전체 변수는 codebook 파일에 별도로 정리하였다. 분석 단위는 “학생 1명과 동문 1명의 후보 매칭쌍”이며, 10명의 학생과 10명의 동문을 기반으로 총 50개의 후보쌍을 구성하였다.

5. 분석 방법

분석 단계 분석 목적 SPSS 메뉴 해석 포인트
1. 데이터 불러오기 학생-동문 후보 매칭 데이터 확인 파일 → 열기 → 데이터 문자형 변수와 숫자형 변수를 구분한다.
2. 빈도분석 학생 관심 산업과 동문 산업의 분포 파악 분석 → 기술통계량 → 빈도분석 관심 수요와 공급이 어떤 산업에 몰려 있는지 본다.
3. 기술통계 매칭점수와 관계강도 수준 확인 분석 → 기술통계량 → 기술통계 평균, 표준편차, 최대/최소로 매칭 품질을 진단한다.
4. 교차분석 관심 산업과 추천 방식의 관계 확인 분석 → 기술통계량 → 교차분석 특정 산업 관심 학생에게 어떤 지원 방식이 필요한지 본다.
5. 상관분석 매칭점수와 예상 도움 정도의 관련성 확인 분석 → 상관분석 → 이변량 상관계수 매칭점수가 높을수록 예상 진로 도움도 커지는지 확인한다.
6. 평균 비교 추천 여부에 따른 매칭점수 차이 검증 분석 → 평균비교 → 독립표본 t 검정 추천 후보와 비추천 후보의 점수 차이가 충분히 큰지 확인한다.
7. 회귀분석 예상 진로 도움 정도를 설명하는 요인 탐색 분석 → 회귀분석 → 선형 무엇이 멘토링 효과를 높이는지 해석한다.
8. 군집분석 학생-멘토 매칭 유형 분류 분석 → 분류분석 → K-평균 군집분석 고적합 1:1형, 그룹멘토링형, 정보세션형으로 분류한다.
9. 그래프 작성 네트워크형 산출물 제작 그래프 → 차트 작성기 상위 매칭쌍과 산업별 부족 분야를 시각화한다.

6. 분석 결과 예시

시범 데이터에서는 AI/Software, Startup/Business, Finance 분야에서 학생 관심이 상대적으로 높게 나타났다. 그러나 동문 공급이 학생 관심과 항상 일치하지는 않았다. 따라서 동문회가 멘토단을 구성할 때 학생 수요가 높은 산업부터 우선 확보할 필요가 있다.

아래 그림은 학생-동문 추천 매칭 네트워크를 시각화한 것이다. 왼쪽은 학생, 오른쪽은 동문이며, 선은 우선 추천된 멘토링 관계를 의미한다.

매칭점수 상위 후보는 대체로 학생의 관심 산업, 희망 직무, 필요 역량과 동문의 현재 직무 및 멘토링 가능 분야가 함께 일치하는 경우였다.

핵심 해석

1:1 멘토링은 모든 학생에게 무조건 제공하기보다 매칭점수와 멘토 가용성을 함께 고려해야 한다.

동문 수가 많은 분야보다 학생 수요가 높은 분야가 우선이다. 데이터는 “누가 도울 수 있는가”와 “누가 도움이 필요한가”를 함께 보여준다.

동문 네트워크는 단순한 인맥이 아니라, 학생 진로 정보 격차를 줄이는 학교 고유의 사회적 자본이다.

8. 한계점과 후속 연구

본 데이터는 시범용 가상 데이터이므로 실제 운영 전 학생과 동문의 자발적 동의가 필요하다.

매칭점수는 산업·전공·직무 중심으로 계산했으나, 실제 멘토링 효과는 성격, 대화 방식, 일정 조율, 신뢰 형성에도 영향을 받는다.

후속 연구에서는 멘토링 전후 진로 확신 변화, 전공 이해도 변화, 포트폴리오 개선 정도를 추적하면 좋다.

네트워크 분석 도구(Gephi, NodeXL, Python networkx)를 활용하면 동문 네트워크의 허브와 구조적 공백을 더 정교하게 분석할 수 있다.

9. 진로·전공 관심과의 연결

이 프로젝트는 경영학의 인적자원관리, 데이터사이언스의 추천 알고리즘, 네트워크 과학의 관계 분석, 교육학의 진로 상담을 연결한다. 학생은 단순히 “멘토를 소개받는 사람”이 아니라, 데이터를 통해 학교 공동체의 진로 지원 구조를 설계하는 문제 해결자가 된다.

10. 에세이형 성찰 문장

나는 처음에 동문 네트워크를 단순한 선배와 후배의 관계로 보았다. 그러나 데이터를 만들고 분석하면서 동문 네트워크가 학생의 진로 정보 격차를 줄이는 학교 고유의 자산이 될 수 있음을 알게 되었다. 중요한 것은 누구를 아는가가 아니라, 필요한 학생에게 적절한 경험과 조언이 공정하게 연결되는 구조를 만드는 일이다.
