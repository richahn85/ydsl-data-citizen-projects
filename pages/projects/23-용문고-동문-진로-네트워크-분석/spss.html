23번 SPSS 분석 가이드: 용문고 동문 진로 네트워크 분석

이 가이드는 23_dataset.csv 또는 23_dataset.xlsx의 Data 시트를 SPSS로 불러와 분석하는 절차를 설명한다.

1. 데이터 불러오기

SPSS 실행 → 파일 → 열기 → 데이터 → 23_dataset.csv 선택

첫 행을 변수 이름으로 읽기 선택

문자형 변수(pair_id, student_id, alumni_id 등)는 명목, 점수형 변수는 척도로 설정

recommended_pair는 0=비추천, 1=우선 추천으로 값 레이블 지정

2. 변수 보기 설정

변수 측도 값 레이블/설명
industry_match 명목 0=불일치, 1=일치
major_match 명목 0=불일치, 1=일치
career_match 명목 0=불일치, 1=일치
matching_score 척도 0~100점
recommended_pair 명목 0=비추천, 1=우선 추천

3. 권장 분석 절차

순서 분석 SPSS 메뉴 핵심 해석
1. 데이터 불러오기 학생-동문 후보 매칭 데이터 확인 파일 → 열기 → 데이터 문자형 변수와 숫자형 변수를 구분한다.
2. 빈도분석 학생 관심 산업과 동문 산업의 분포 파악 분석 → 기술통계량 → 빈도분석 관심 수요와 공급이 어떤 산업에 몰려 있는지 본다.
3. 기술통계 매칭점수와 관계강도 수준 확인 분석 → 기술통계량 → 기술통계 평균, 표준편차, 최대/최소로 매칭 품질을 진단한다.
4. 교차분석 관심 산업과 추천 방식의 관계 확인 분석 → 기술통계량 → 교차분석 특정 산업 관심 학생에게 어떤 지원 방식이 필요한지 본다.
5. 상관분석 매칭점수와 예상 도움 정도의 관련성 확인 분석 → 상관분석 → 이변량 상관계수 매칭점수가 높을수록 예상 진로 도움도 커지는지 확인한다.
6. 평균 비교 추천 여부에 따른 매칭점수 차이 검증 분석 → 평균비교 → 독립표본 t 검정 추천 후보와 비추천 후보의 점수 차이가 충분히 큰지 확인한다.
7. 회귀분석 예상 진로 도움 정도를 설명하는 요인 탐색 분석 → 회귀분석 → 선형 무엇이 멘토링 효과를 높이는지 해석한다.
8. 군집분석 학생-멘토 매칭 유형 분류 분석 → 분류분석 → K-평균 군집분석 고적합 1:1형, 그룹멘토링형, 정보세션형으로 분류한다.
9. 그래프 작성 네트워크형 산출물 제작 그래프 → 차트 작성기 상위 매칭쌍과 산업별 부족 분야를 시각화한다.

4. 결과 해석 문장 예시

추천 매칭쌍의 평균 매칭점수가 비추천 후보보다 높게 나타나면, 알고리즘이 학생-동문 적합도를 일정 부분 반영한다고 볼 수 있다.

학생 관심 산업이 높은데 동문 공급이 낮은 분야는 동문회가 신규 멘토를 우선 확보해야 할 영역이다.

mentor_network_degree와 expected_career_help가 양의 상관을 보이면 연결도가 높은 동문이 학생에게 더 넓은 진로 정보를 제공할 가능성이 있다.

단, 멘토링은 사람이 사람을 만나는 과정이므로 수치만으로 최종 배정을 확정하지 않고 학생 희망과 동문 의사를 함께 확인해야 한다.

5. 보고서에 넣을 표와 그래프

산출물 작성 방법 보고서 해석
산업별 수요-공급 표 Summary 시트 활용 학생 관심과 동문 멘토 공급의 불균형 확인
상위 매칭쌍 그래프 matching_score 상위 10개 정렬 우선 연결할 멘토 후보 제시
추천 방식 빈도표 recommended_action 빈도분석 1:1/그룹/정보세션 비율 확인
네트워크 지도 학생-동문 추천쌍 선으로 연결 동문 네트워크의 구조적 활용 가능성 제시

* Project 23: Yongmoon Alumni Career Network Analysis.
GET DATA
  /TYPE=TXT
  /FILE='23_dataset.csv'
  /ENCODING='UTF8'
  /DELCASE=LINE
  /DELIMITERS=','
  /QUALIFIER='"'
  /ARRANGEMENT=DELIMITED
  /FIRSTCASE=2
  /VARIABLES=
    pair_id A8 student_id A8 grade F1.0 desired_major A30 career_interest A30 industry_interest A30
    career_clarity F1.0 mentoring_need F1.0 skill_ai F1.0 skill_data F1.0 skill_business F1.0 skill_communication F1.0
    alumni_id A8 alumni_cohort F2.0 alumni_major A30 alumni_industry A30 alumni_job_field A30 mentoring_area A60
    monthly_available_hours F2.0 online_possible F1.0 mentor_network_degree F2.0 industry_match F1.0 major_match F1.0 career_match F1.0
    skill_match_score F5.1 availability_score F4.1 matching_score F5.1 tie_strength_score F5.1 expected_career_help F3.1
    student_match_rank F1.0 recommended_pair F1.0 recommended_action A30.
EXECUTE.

VARIABLE LABELS
 matching_score 'Student-alumni matching fit score (0-100)'
 tie_strength_score 'Expected tie strength score (0-100)'
 expected_career_help 'Expected career help (1-5)'
 recommended_pair 'Priority recommended mentoring pair'.
VALUE LABELS recommended_pair 0 'Not priority' 1 'Priority recommended'.
VALUE LABELS online_possible 0 'No' 1 'Yes'.
VALUE LABELS industry_match major_match career_match 0 'No match' 1 'Match'.

FREQUENCIES VARIABLES=industry_interest alumni_industry recommended_action recommended_pair.
DESCRIPTIVES VARIABLES=matching_score tie_strength_score expected_career_help mentor_network_degree mentoring_need.
CROSSTABS /TABLES=industry_interest BY recommended_action /CELLS=COUNT ROW COLUMN.
CORRELATIONS /VARIABLES=matching_score tie_strength_score expected_career_help mentor_network_degree mentoring_need career_clarity.
T-TEST GROUPS=recommended_pair(0 1) /VARIABLES=matching_score expected_career_help tie_strength_score.
REGRESSION /DEPENDENT expected_career_help /METHOD=ENTER matching_score mentor_network_degree career_clarity mentoring_need.
QUICK CLUSTER matching_score tie_strength_score mentor_network_degree mentoring_need /CRITERIA=CLUSTER(3) MXITER(10) /SAVE CLUSTER.
