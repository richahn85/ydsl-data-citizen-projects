  1. 독거노인 생활 인프라 접근성 분석

병원·약국·복지관·대중교통 접근성 기반 생활 인프라 취약지역 지도

1. 핵심 요약

본 보고서는 성북구와 용문고 주변 생활권을 대상으로 독거노인이 병원, 약국, 노인복지시설, 버스정류장, 지하철역 등 생활 필수 인프라에 얼마나 쉽게 접근할 수 있는지를 분석한 시범 보고서이다. 시범 데이터 50개 생활권 분석 결과 평균 접근성 점수는 42.2점, 평균 개선 우선순위 점수는 52.1점으로 나타났으며, 높음 이상 우선 개선 대상은 18개 생활권이었다.

이 프로젝트의 핵심은 독거노인의 어려움을 개인의 이동 능력 부족으로만 보지 않고, 생활 인프라의 위치와 보행환경이라는 구조적 조건을 함께 보는 데 있다. 접근성이 낮은 생활권은 병원이나 약국이 멀고, 버스정류장까지의 거리도 길며, 경사·좁은 보도·쉼터 부족이 겹쳐 일상생활의 불편이 커질 가능성이 높다.

2. 문제 정의 카드

질문 답변
내가 본 문제는 무엇인가? 독거노인이 병원, 약국, 복지관, 대중교통에 쉽게 접근하지 못하는 생활권이 있을 수 있다.
왜 이 문제가 중요한가? 생활 인프라 접근성은 건강관리, 사회적 고립 완화, 응급상황 대응과 직결되기 때문이다.
어떤 데이터를 사용할 것인가? 독거노인 분포, 병의원·약국·노인복지시설 위치, 버스정류장·지하철역 위치, 보도 폭, 경사도, 현장조사 데이터.
어떤 분석을 할 것인가? 거리 분석, 접근성 점수화, 취약성 점수화, 개선 우선순위 산출, 지도 시각화, 군집분석.
누구에게 제안할 것인가? 성북구청 어르신복지과, 교통행정과, 보건소, 동주민센터, 지역 복지관.
전공 관심과 어떻게 연결되는가? 사회복지, 도시계획, 공공보건, 데이터사이언스가 결합된 사람 중심 도시 문제 해결 경험이 된다.

3. 데이터 설계

분석 단위는 생활권 또는 250~500m 격자 1개이다. 실제 프로젝트에서는 행정동 단위 독거노인 수와 시설 좌표를 결합하고, 학생이 현장조사로 보행환경을 보완한다. 본 파일의 데이터는 공식 공공데이터 구조를 반영한 시범용 가상 데이터이다.

5. 분석 방법

분석 단계 분석 목적 사용 변수 SPSS 메뉴 경로 해석 포인트
1. 데이터 불러오기 CSV/XLSX를 SPSS로 불러오고 변수 측도 지정 전체 변수 파일 → 열기 → 데이터 문자형과 숫자형, 명목·서열·척도 구분
2. 기술통계 접근성·취약성·우선순위 점수의 분포 확인 accessibility_score, vulnerability_score, improvement_priority_score 분석 → 기술통계량 → 기술통계 평균·표준편차·최소·최대값 확인
3. 빈도분석 개선 우선순위 등급별 구간 수 확인 priority_level, cluster_type 분석 → 기술통계량 → 빈도분석 매우 높음·높음 구간 비중 확인
4. 상관분석 접근성 점수와 개선 우선순위의 관계 확인 accessibility_score, vulnerability_score, improvement_priority_score, slope_score_1_5, sidewalk_width_m 분석 → 상관분석 → 이변량 상관계수 접근성이 낮을수록 우선순위가 높은지 확인
5. 회귀분석 어떤 요인이 개선 우선순위를 설명하는지 분석 improvement_priority_score / hospital_distance_m, pharmacy_distance_m, welfare_distance_m, bus_stop_distance_m, slope_score_1_5, sidewalk_width_m 분석 → 회귀분석 → 선형 표준화계수 Beta와 유의확률 해석
6. 군집분석 생활권을 취약 유형별로 분류 facility_score, distance_score, environment_score, vulnerability_score 분석 → 분류분석 → K-평균 군집분석 의료접근 취약형, 교통접근 취약형 등 유형화
7. 지도 시각화 취약 생활권 위치를 지도형으로 표현 x_m, y_m, improvement_priority_score, priority_level 그래프 → 차트 작성기 → 산점도 점 크기 또는 색상으로 우선순위 표현

8. 한계점과 후속 연구

본 분석은 시범용 가상 데이터에 기반하므로 실제 정책 제안 전에는 최신 공공데이터와 현장조사를 결합해야 한다. 특히 독거노인 위치는 개인정보 보호가 중요하므로 개인 주소가 아니라 행정동 또는 격자 단위의 집계자료를 사용해야 한다. 후속 연구에서는 실제 보행 네트워크 거리, 경사도, 엘리베이터·에스컬레이터 위치, 야간 조명, 무더위쉼터, 병원 진료시간까지 결합하면 더 현실적인 접근성 모델을 만들 수 있다.

9. 진로·전공 관심과의 연결

이 프로젝트는 데이터사이언스와 사회복지를 연결해 독거노인을 숫자가 아니라 생활 조건 속의 사람으로 바라보게 한다. 학생은 단순히 시설의 개수를 세는 데서 그치지 않고, 누가 어떤 길을 걷고, 어디에서 멈추고, 어떤 도움을 필요로 하는지 질문하게 된다. 이는 사람 중심의 도시계획, 공공보건, 복지정책으로 확장될 수 있다.

10. 에세이형 성찰 문장

나는 처음에 병원과 약국이 동네에 있으면 충분하다고 생각했다. 그러나 데이터를 통해 같은 시설이라도 독거노인이 실제로 걸어갈 수 있는 거리인지, 경사가 심하지 않은지, 버스정류장과 연결되는지에 따라 접근성이 달라진다는 것을 알게 되었다. 이 분석은 복지를 시설의 개수로만 보는 것이 아니라, 사람의 이동과 생활의 관점에서 보아야 한다는 깨달음을 주었다.
